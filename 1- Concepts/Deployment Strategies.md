
Deployment Strategies are approaches used to **release new application versions while managing availability, risk, traffic distribution, and rollback behavior** during software deployment.

Different strategies balance trade-offs between reliability, deployment speed, infrastructure cost, and user impact.

---
### `Key Ideas`

- **Controlled application rollout**  
    Deployment strategies define how new application versions replace or coexist with existing versions during deployment.

- **Risk management**  
    Different strategies reduce deployment risk by controlling traffic exposure, enabling rollback mechanisms, or isolating failures.

- **Availability and reliability**  
    Modern deployment systems aim to minimize downtime and service disruption while transitioning between versions.

- **Traffic and version coordination**  
    Some strategies distribute traffic across multiple application versions simultaneously to support testing, gradual rollout, or validation.

- **Common strategies** 
    - Recreate
    - Rolling Deployment  
    - Blue/Green Deployment
    - Canary Deployment  
    - A/B Testing
    - Shadow Deployment
