
**Cost-Based Optimizer** is the mechanism that evaluates alternative query execution plans and selects the most efficient one based on estimated resource costs. It chooses a plan by comparing predicted performance metrics such as I/O, CPU usage, and memory consumption.

---
### `How It Works`

- **Receives Candidate Plans**  
    Takes execution strategies generated by the query planner.

- **Estimates Resource Costs**  
    Uses statistical information (e.g., data distribution, table sizes, index selectivity) to estimate the cost of each plan.

- **Compares Alternatives**  
    Evaluates different join orders, access paths, and operator choices.

- **Selects the Optimal Plan**  
    Chooses the plan with the lowest estimated cost for execution.

- **Relies on Statistics**  
    Depends on maintained metadata and statistics about stored data to make informed decisions.

---
### `Why It Exists`

- To improve query performance automatically
- To reduce unnecessary I/O and computation
- To adapt execution strategies to data size and distribution
- To avoid inefficient execution paths
