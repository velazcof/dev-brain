
**Object-Relational Mappers (ORMs)** are software libraries that map objects in application code to relational database tables. They allow developers to interact with relational databases using programming language constructs instead of writing raw SQL.

---
### `Key Capabilities`

- **Object-to-Table Mapping**  
    Maps classes to tables and object properties to columns.
    
- **Query Abstraction**  
    Provides APIs or query builders to construct database queries programmatically.
    
- **Change Tracking**  
    Detects modifications to objects and translates them into database updates.
    
- **Relationship Handling**  
    Manages associations such as one-to-many and many-to-many relationships.
    
- **Schema Management**  
    Often supports migrations and automated schema generation.

---
### `Use Cases`

- Backend application development
    
- Rapid application prototyping
    
- Reducing repetitive SQL boilerplate
    
- Maintaining database access consistency in large codebases
