
A Container Diagram shows the major building blocks of a software system—called containers—such as web apps, APIs, databases, queues, and mobile apps. It illustrates how these containers interact and what responsibilities each one has.

---
### `Key Ideas`

- Provides a **mid-level architectural view** (zoomed in from system context).
    
- Shows containers like:
    
    - Web frontend
    - Backend services
    - Databases
    - Message brokers
    - External APIs
    
- Describes responsibilities, technology choices, and communication paths.
    
- Helps developers understand the system’s main deployable units.
